1. git add      – adaugă fișierele modificate în staging area
2. git clone    – copiază un repository Git existent pe calculator
3. git commit   – salvează schimbările pregătite într-o nouă versiune
4. git push     – trimite commit-urile locale către repository-ul de pe server
5. git status   – arată starea fișierelor: modificate, adăugate sau neînregistrate
6. git log      – afișează istoricul commit-urilor din repository
7. git fetch    – aduce actualizările de pe server fără a le îmbina cu codul local
