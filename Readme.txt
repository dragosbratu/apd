   1. git add - adaugă fișierele modificate în zona de staging (pregătire) pentru următorul commit
   2. git clone - copiază un depozit (repository) existent de pe un server pe calculatorul meu
   3. git commit - salvează modificările din staging area în istoria locală a proiectului, cu un mesaj descriptiv
   4. git push - Trimite commit-urile locale către un repository remote
   5. git status - Arată starea curentă a repository-ului: ce fișiere sunt modificate, ce e în staging, ce nu e tracked etc
   6. git log - Afișează istoricul commit-urilor: cine a făcut ce modificări, când și cu ce mesaj
   7. git fetch - Descarcă ultimele modificări de pe repository-ul remote, dar nu le îmbină automat cu codul meu
